# 题型说明

## Special Judge（校验器）

识别文件名：`checker` 或 `chk`。

校验器使用 testlib 标准，实现于 `jury.jury` 函数中。

校验器以如下命令行参数启动：

```sh
./checker <infile> <outfile> <ansfile>
```

其中 `<infile>` `<outfile>` `<ansfile>` 分别是题目输入文件、选手输出文件、题目答案文件。

校验器运行失败时，评测附加信息会显示校验器的运行状态；否则评测附加信息为校验器向标准错误流输出的第一行。

你可以选择不使用 testlib 并自行实现校验器，但是其命令行参数、返回值、输出格式必须与 testlib 相同。

## IO 交互题（交互库）

识别文件名：`interactor`。

IO 交互题使用 testlib 标准，实现于 `sandbox.run_interactive` 函数中。

交互库以如下命令行参数启动：

```sh
./interactor <infile> <outfile>
```

其中 `<infile>` `<outfile>` 分别是题目输入文件和交互库输出文件。交互库需要从标准输入流读取选手输出的信息，向标准输出流写入信息。

交互库运行失败时，评测附加信息会显示交互库的运行状态。

IO 交互题仍然可以使用校验器，如果交互库和选手程序都运行成功，交互库向 testlib tlog 流输出的内容将作为校验器使用的“选手输出文件”。

你可以选择不使用 testlib 并自行实现交互库，但是其命令行参数、返回值、输出格式必须与 testlib 相同。

评测方式：分别启动两个沙箱运行选手程序和交互库，通过管道连接输入输出，主程序持有管道端口，使用两个中继器分别并行地连接两组管道。管道连接完毕后启动子程序直到运行结束。

## 函数式交互题

识别文件名：`grader`。

函数式交互题仅支持 C++ 语言，实现于 `jury.compile_program` 函数中，该题型对评测行为的影响也仅限于该函数，其本质是多文件编译。

如果需要添加其它语言的函数式交互支持，请修改该函数。

## 提交答案题

暂时不支持。

## 通信题

暂时不支持。
